Plaud CLI lets you manage your Plaud recordings from the terminal — browse your recording list, get file details, extract transcripts and summaries, download audio, and manage account authorization.

---

## Prerequisites

- Node.js >= 20
- A Plaud account

---

## Installation

```bash
npm install -g @plaud-ai/cli
```

---

## Quick Start

**1. Log in**

```bash
plaud login
```

Your browser will open the authorization page automatically. Once authorized, the terminal displays a login success message.

**2. Verify login status**

```bash
plaud me
```

**3. List your recordings**

```bash
plaud files
```

**4. Extract a transcript**

```bash
plaud transcript <file_id>
```

**5. View an AI summary**

```bash
plaud summary <file_id>
```

## Command Reference

### `plaud login`

Opens your browser to complete OAuth authorization. Tokens are saved locally.

```bash
plaud login
```

### `plaud logout`

Logs out and revokes authorization.

```bash
plaud logout
```

### `plaud version`

Displays version, commit hash, and build time.

```bash
plaud version
```

Example output:

```
plaud 0.1.2
commit  a3f9c12
built   2026-04-14T08:30:00Z
```

### `plaud me`

Displays current account information.

```bash
plaud me
```

### `plaud files`

Lists your recording files.

```bash
plaud files
plaud files --page <number> --page-size <number>
```

| Option | Description | Default |
|---|---|---|
| `-p, --page` | Page number (1–1000) | `1` |
| `-s, --page-size` | Items per page (10–100) | `20` |

**Examples**

```bash
plaud files --page 2 --page-size 10
plaud files -p 2 -s 10
```

### `plaud file <file_id>`

Displays detailed information for a single recording.

```bash
plaud file <file_id>
```

**Example**

```bash
plaud file 4ee917c43f013423df146b97fd14610f
```

### `plaud audio <file_id>`

Returns a temporary audio download URL for the recording (valid for 24 hours).

```bash
plaud audio <file_id>
```

**Example**

```bash
plaud audio 4ee917c43f013423df146b97fd14610f
```

### `plaud transcript <file_id>`

Extracts a sentence-by-sentence transcript, formatted as `[MM:SS - MM:SS] Speaker: Content`.

```bash
plaud transcript <file_id>
plaud transcript <file_id> --output <file>
```

| Option | Description |
|---|---|
| `-o, --output <file>` | Save the transcript to the specified file |

**Examples**

```bash
plaud transcript 4ee917c43f013423df146b97fd14610f
plaud transcript 4ee917c43f013423df146b97fd14610f -o transcript.txt
```

### `plaud summary <file_id>`

Returns the AI-generated summary for a recording (Markdown format).

```bash
plaud summary <file_id>
plaud summary <file_id> --output <file>
```

| Option | Description |
|---|---|
| `-o, --output <file>` | Save the summary to the specified file |

**Examples**

```bash
plaud summary 4ee917c43f013423df146b97fd14610f
plaud summary 4ee917c43f013423df146b97fd14610f -o summary.md
```

## Data Reference

### Recording File Fields

The following fields are returned in both the list (`plaud files`) and detail (`plaud file <id>`) views:

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique file identifier |
| `name` | string | File name |
| `created_at` | string | Creation time (ISO 8601) |
| `duration` | string | Recording duration (list: `Xmin`; detail: `Xmin Ysec` or `Ysec`) |

The following fields are returned only in the detail view (`plaud file <id>`):

| Field | Type | Description |
|---|---|---|
| `start_at` | string | Recording start time (ISO 8601) |
| `serial_number` | string | Serial number of the device that made the recording |
| `audio` | string | Audio availability (`available` / `unavailable`) |
| `transcript` | string | Transcript availability (`available` / `unavailable`) |
| `summary` | string | AI summary availability (`available` / `unavailable`) |

### Local Storage

Tokens are stored in `~/.plaud/tokens.json` and managed automatically by the CLI. No manual edits needed.

### Configuration File

Optional config file at `~/.plaud/cli.yaml`. Supported fields:

```yaml
api_base: "https://platform.plaud.ai/developer/api"
auth_url: "https://app.plaud.ai/platform/oauth"
timeout: 30000   # ms
```

Priority order: **environment variable > `~/.plaud/cli.yaml` > built-in default**.

### Exit Codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | Invalid arguments or unknown error |
| `2` | Authentication failed (token invalid or expired) |
| `3` | Server unreachable (network error) |
| `4` | Request timed out |

### Error Output Format

All errors are written to **stderr** in a structured format, leaving stdout clean for scripting:

```
✖ [AUTH_FAILED] Token invalid or expired. Run `plaud login`.
```

```bash
npm uninstall -g @plaud-ai/cli

```

## FAQ

**What do I do if my token expires?**

The CLI refreshes tokens automatically. If refresh fails, run `plaud login` again.
