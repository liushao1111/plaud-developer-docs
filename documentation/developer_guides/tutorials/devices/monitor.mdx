This guide will show you how to monitor a specific device's status and health metrics using the Device Monitoring API. 
You'll need a `device_id` from the previous binding step.

## Using the Plaud API

<Steps>

<Step title="Create an API Token">

Create an API token using your [client credentials](/api-guide/api-intro/authorization), which you'll use to securely access the API.

```
PLAUD_API_TOKEN = <your_api_token_here>
```

</Step>

<Step title="Make the API request">

Monitor a specific device's status and health metrics using its device_id:

<CodeGroup>

```python Python
import requests

BASE_URL = "https://platform.plaud.cn/api"
headers = {
    "Authorization": f"Bearer {PLAUD_API_TOKEN}",
    "Content-Type": "application/json"
}

# Use device_id from the binding step
device_id = <your_device_id>  # Replace with your device_id

response = requests.get(
    f"{BASE_URL}/devices/{device_id}",
    headers=headers
)

if response.status_code == 200:
    device = response.json()
    print(f"✅ Device found: {device.get('name', device['id'][:8])}")
    print(f"Battery: {device.get('battery')}%")
    print(f"Storage: {device.get('free_space')}MB available")
else:
    print(f"❌ Error: {response.status_code}")
```

```curl cURL
curl -X GET "https://platform.plaud.cn/api/devices/dvc_abc123def456" \
  -H "Authorization: Bearer $PLAUD_API_TOKEN"
```

</CodeGroup>

<Tip>
For detailed response structure, see the [Get Device API reference](/api-reference/device/get-device-by-id).
</Tip>

</Step>

<Step title="Execute the Code">

<CodeGroup>
    ```python Python
    python example.py
    ```
</CodeGroup>
You should see the health information printed to the console.

</Step>

</Steps>

## Monitoring Metrics

| Metric | Description | Alert Threshold |
|:-------|:------------|:----------------|
| **Battery Level** | Device battery percentage | < 20% |
| **Storage Usage** | Used storage percentage | > 90% |
| **WiFi Signal** | WiFi signal strength (dBm) | < -70 dBm |
| **Last Update** | Time since last status update | > 24 hours |
| **Binding Status** | Whether device is bound to user | Unbound devices |

## Common Error Scenarios

| Error Code | Cause |
|:-----------|:------|
| **Device Not Found** | Device doesn't exist in system, returns `404` |
| **Invalid Status Data** | Malformed status update, returns `422` validation error |
| **Rate Limit Exceeded** | Too many status updates, returns `429` |
