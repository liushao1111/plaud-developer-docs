## Overview

Webhooks allow you to receive automatic notifications when file-related events occur in your Plaud applications, such as:

- File upload completion
- File metadata updates
- File deletion

When a file event occurs, Plaud will send a POST request to your specified webhook URL with the event data, enabling you to build real-time, event-driven applications. 
For other event types, see the [Webhook Event Reference](/documentation/developer_guides/webhook_events).

## Using webhooks

<Steps>

<Step title="Create a Webhook Endpoint">

First, you'll need to create a publicly accessible HTTPS endpoint to receive webhook notifications.

<CodeGroup>

```python Python
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/webhook/plaud', methods=['POST'])
def handle_plaud_webhook():
    # Parse the event data
    event_data = request.get_json()
    event_type = event_data.get('event_type')
    
    # Handle file event types
    if event_type == 'file.created':
        file_info = event_data.get('data', {})
        print(f"File uploaded: {file_info.get('name')}")
    elif event_type == 'file.updated':
        file_info = event_data.get('data', {})
        print(f"File updated: {file_info.get('id')}")
    
    return jsonify({'received': True}), 200

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=3000)
```

```curl cURL
curl -X POST "https://your-domain.com/webhook/plaud" \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "file.created",
    "data": {
      "id": "file_abc123def456",
      "name": "meeting_recording.opus"
    }
  }'
```

</CodeGroup>

Your webhook endpoint is ready to receive notifications.

</Step>

<Step title="Configure Webhook URL">

Configure your webhook URL in the Plaud platform through your client settings or dashboard to subscribe to file events.

```python Python
# Example webhook configuration
webhook_url = "https://your-domain.com/webhook/plaud"
events = ["file.created", "file.updated", "file.deleted"]
```

</Step>

<Step title="Test Your Webhook">

Test your webhook by uploading a file, which will trigger a `file.created` event:

</Step>

</Steps>

## File Webhook Events

| Event Type | Description | Payload |
|:-----------|:------------|:--------|
| **file.created** | File upload completed successfully | File information with ID, name, owner, device |
| **file.updated** | File metadata updated | Updated file information |
| **file.deleted** | File deleted from system | File ID and deletion timestamp |

## Webhook Payload Structure

All webhook payloads follow this consistent structure:

```json
{
  "event_type": "file.created",
  "timestamp": 1703980800,
  "event_id": "evt_abc123def456",
  "data": {
    // Event-specific data
  }
}
```

### Example: File Created Payload
```json
{
  "event_type": "file.created",
  "timestamp": 1703980800,
  "event_id": "evt_file_001",
  "data": {
    "id": "file_abc123def456",
    "name": "meeting_recording.opus",
    "owner_id": "user_12345",
    "device_id": "dvc_notepin_001",
    "filesize": 2048576,
    "duration": 180000,
    "created_at": "2024-01-01T10:00:00Z"
  }
}
```

## Best Practices

- **Return 200**: Always return a `200 OK` response to acknowledge receipt of a webhook.
- **Keep it fast**: Process webhooks quickly. If you need to perform long-running tasks, do so asynchronously.
- **Use HTTPS**: Your webhook endpoint URL must use HTTPS for security.
- **Handle duplicates**: Design your endpoint to be idempotent in case of duplicate event deliveries.
