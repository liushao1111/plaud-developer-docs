## Overview

Workflow automation tools like **n8n** and **Zapier** excel at orchestrating different services into reusable, multi-step tasks. 
For more general and diverse post-recording processing, Plaud offers powerful, customizable workflows designed to transform raw audio into structured, actionable data.

By using Plaud's workflow capabilities, you can:
- **Automate End-to-End Processing**: Save significant manual effort by automating the entire pipeline from raw audio to structured insights.
- **Ensure Data Consistency**: Standardize processes with repeatable workflows for reliable and consistent data extraction.
- **Build Custom Solutions**: Create bespoke pipelines by combining Plaud's core AI services with your own business logic.

## Use Case: Healthcare Automation

<Frame>
  <img src="/assets/workflow.png" alt="Workflow Flowchart" caption="A Healthcare Workflow Usecase"/>
</Frame>

In many healthcare settings, such as clinics and hospitals, practitioners handle numerous patient consultations daily. A significant amount of time is spent manually documenting these interactions. This use case demonstrates how a Plaud workflow can fully automate this process, transforming a long audio file of back-to-back patient visits into structured, actionable clinical notes.

<Steps>
  <Step title="1. From Voice to Text">
    The workflow starts with a single long audio file containing multiple patient consultations. The first step is to accurately transcribe the entire recording into text, creating a complete written record of all conversations.
  </Step>
  <Step title="2. Identifying Individual Consultations">
    Manually separating conversations is tedious. The workflow automates this by intelligently identifying where one consultation ends and another begins, using cues like speaker changes to split the long transcript into separate segments for each patient.
  </Step>
  <Step title="3. Extracting Key Medical Information">
    For each patient's transcript, the workflow acts like a highly-trained assistant. It reads the text and pulls out critical information based on a pre-defined template. This could include the patient's chief complaint, prescribed treatments, and other important details. The result is organized, structured data, much like filling out a digital form automatically.
    <Info>
      **For Developers**: The extracted data is delivered as a clean JSON object based on your schema. This structured format allows for easy integration into Electronic Health Record (EHR) systems.
      ```json
      {
        "chief_complaint": {
          "item": "Persistent headaches",
          "description": "Patient reports daily headaches for the past two weeks, primarily in the morning."
        },
        "treatments": {
          "items": ["Pain relievers"],
          "description": "Currently taking over-the-counter pain relievers with little effect."
        }
      }
      ```
    </Info>
  </Step>
  <Step title="4. Generating Reports and Sending Alerts">
    Finally, the structured information from all patients is compiled into a comprehensive summary report, providing a clear overview of all consultations. Once the report is ready, an automatic notification is sent, ensuring the medical team can review the information promptly.
  </Step>
</Steps>

## FAQ

<AccordionGroup>

  <Accordion title="How is Intelligent Extraction different from a summary?">
    A summary condenses a conversation into a brief, narrative overview of the key points. In contrast, Intelligent Extraction (AI ETL) pulls out specific, discrete data points and organizes them into a structured format like JSON. A summary gives you the gist, while extraction gives you actionable data fields.
  </Accordion>

  <Accordion title="Can I create my own extraction schemas?">
    Yes. While Plaud provides a set of pre-built templates for common industries, you have the flexibility to define fully custom schemas tailored to your unique requirements. This ensures you can extract precisely the information you need for any use case.
  </Accordion>

  <Accordion title="How do I get the results of my workflow?">
    AI Workflows run asynchronously in the background. Once you submit a workflow, you can periodically check its status using our API. The recommended method is to use Webhooks, which will automatically send a notification to your server with the results as soon as the workflow is complete.
  </Accordion>
</AccordionGroup>
