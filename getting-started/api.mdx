## Step 1: Authentication

All API requests require authentication using your API key in the Authorization header:

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.plaud.ai/files/
```

<div className="best-practices">
  <h3>🔒 Security Best Practices</h3>
  <ul>
    <li>Never expose your API key in client-side code</li>
    <li>Use environment variables to store your API key</li>
    <li>Rotate your API keys regularly</li>
  </ul>
</div>

## Step 4: Working with AI Features

One of Plaud's key features is AI-powered transcription and summarization. Here's how to access these features:

### Get File with AI Results

```bash
curl -H "Authorization: YOUR_API_KEY" \
  https://api.plaud.ai/files/file_123
```

### Check AI Processing Status

The `ai_status` field indicates the processing state:
- `pending` - AI processing queued
- `processing` - AI processing in progress  
- `completed` - AI processing finished
- `failed` - AI processing failed

### Access AI Results

```json
{
  "ai_result_list": [
    {
      "type": "transcription",
      "status": "completed",
      "result": "Full transcription text here..."
    },
    {
      "type": "summary", 
      "status": "completed",
      "result": "AI-generated summary here..."
    }
  ]
}
```

## Step 5: Error Handling

The API uses standard HTTP status codes and returns detailed error information:

```json
{
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid or has expired",
    "details": "Please check your API key and try again"
  }
}
```

<div className="best-practices">
  <h3>🛡️ Common Error Codes</h3>
  <ul>
    <li><strong>401 Unauthorized</strong> - Invalid or missing API key</li>
    <li><strong>403 Forbidden</strong> - Insufficient permissions</li>
    <li><strong>404 Not Found</strong> - Resource not found</li>
    <li><strong>429 Too Many Requests</strong> - Rate limit exceeded</li>
    <li><strong>500 Internal Server Error</strong> - Server error</li>
  </ul>
</div>
